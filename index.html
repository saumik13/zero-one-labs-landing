sisypho

Open source SDK for natural language RPA and custom CUA tools

Sisypho transforms natural language instructions into executable automation workflows. Build deterministic RPA flows that humans can run directly, or create custom tools for more reliable and faster CUA agents.

Quick Start

Installation

pip install sisypho

CLI Usage

# Create a workflow with natural language
python -m sisypho create --task "open chrome and type hello"

# Create with recording
python -m sisypho create --task "open chrome and type hello" --record

# Run a workflow
python -m sisypho run --workflow workflow.json

Example Usage

Python API

import asyncio
from sisypho.utils import RecorderContext, await_task_completion, RpaWorkflow, CuaTool

async def main():
    # Define your automation task in natural language
    task_prompt = """Open a CSV file from Downloads in Microsoft Excel, 
    calculate the total sales column, and paste the result into a new Apple Notes entry. 
    Use deterministic steps without relying on screen coordinates."""

    with RecorderContext() as recorder:
        await_task_completion()

    recording = recorder.get_recording()
    workflow = RpaWorkflow(recording, task_prompt)
    
    await workflow.generate_code()
    result = workflow.run_workflow()
    workflow.save()

Command Line

Core Features

Accessibility-Based Automation

Uses UI accessibility APIs and semantic understanding instead of fragile coordinates or selectors

Deterministic Workflows

Generate reliable, reproducible automation sequences

MCP Protocol Integration

Automatically expose workflows as MCP tools that computer use agents can discover and execute

Semantic Recording

Record user interactions and convert them to accessibility-based workflows that work across UI changes

Custom UI Tools for Computer Use Agents

Sisypho SDK enables you to create reusable UI automation tools that computer use agents can leverage through the MCP protocol. Workflows automatically become powerful automation tools.

Workflow-to-Tool Conversion

When you launch the MCP server, Sisypho automatically:

  • Scans your workflow directory for .json workflow files
  • Registers each workflow as an MCP tool with its task description
  • Exposes the tools via the MCP protocol for agent consumption
# Directory structure
./automation-tools/
├── gmail_automation.json     # Becomes "run_workflow_0" tool
├── slack_notifications.json  # Becomes "run_workflow_1" tool
└── data_entry.json          # Becomes "run_workflow_2" tool

# Launch MCP server
python -m sisypho mcp --workflow-directory ./automation-tools

Agent Integration

Computer use agents can discover and execute your custom UI tools:

# Agent discovers available tools
tools = await mcp_client.list_tools()
# Returns: [
#   {"name": "run_workflow_0", "description": "Automate Gmail inbox management"},
#   {"name": "run_workflow_1", "description": "Send Slack status updates"},
#   {"name": "run_workflow_2", "description": "Fill customer data forms"}
# ]

# Agent executes UI automation
result = await mcp_client.call_tool("run_workflow_0", {})
# Sisypho performs the recorded Gmail automation workflow

create

Create new automation workflows from natural language descriptions

# Basic workflow creation
python -m sisypho create --task "open notepad and write hello world"

# Create with user interaction recording
python -m sisypho create --task "fill out contact form" --record

run

Execute existing workflows or start interactive sessions

# Run a saved workflow
python -m sisypho run --workflow my_workflow.json

# Interactive mode for testing
python -m sisypho run --interactive

mcp

Launch MCP server to expose workflows as tools for computer use agents

# Launch MCP server with workflow directory
python -m sisypho mcp --workflow-directory ./automation-tools

# Workflows become discoverable tools for agents
# Each .json file becomes an executable MCP tool

Python API Reference

RecorderContext

Context manager for capturing user interactions

with RecorderContext() as recorder:
    # Your interactions are recorded here
    recording = recorder.get_recording()

RpaWorkflow

Main workflow orchestrator for automation tasks

workflow = RpaWorkflow(recording, task_prompt)
await workflow.generate_code()
result = workflow.run_workflow()

CuaTool

Tools for integrating deterministic workflows with CUA agents

cua_tools = CuaTool(recording, task_prompt)
# Use with your own CUA implementation
